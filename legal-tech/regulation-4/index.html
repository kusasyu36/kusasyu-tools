はじめに

5日間メール講座「規制変更を見逃さない仕組みの作り方」で解説した考え方を、実際のシステムとして立ち上げるための実装ガイドです。

当社が2025年に実施した国内上場企業30社へのヒアリング調査では、規制監視に金融業で月間約120時間、製薬業で約90時間、製造業で約70時間が費やされていました。本ガイドの仕組みを導入することで、この工数の60〜70%を自動化し、年間平均600件超の改正・通達の見落としリスクを大幅に低減できます。

2-2. 実装手順

① RSSフィード取得スクリプト（Python）

import feedparser
import json
from datetime import datetime

# Step1で特定した優先度Aのソースを登録
FEEDS = [
    {"name": "金融庁", "url": "https://www.fsa.go.jp/rss/news.rdf", "sector": "金融"},
    {"name": "PMDA",  "url": "https://www.pmda.go.jp/rss/...",    "sector": "製薬"},
    # 優先度Aのソースを追加
]

def fetch_new_entries(feeds, last_check):
    new_entries = []
    for feed_conf in feeds:
        feed = feedparser.parse(feed_conf["url"])
        for entry in feed.entries:
            pub_date = datetime(*entry.published_parsed[:6])
            if pub_date > last_check:
                new_entries.append({
                    "source": feed_conf["name"],
                    "sector": feed_conf["sector"],
                    "title": entry.title,
                    "link": entry.link,
                    "published": pub_date.isoformat(),
                    "summary_raw": entry.get("summary", ""),
                })
    return new_entries

② LLM APIによる要約＋影響度スコアリング

from openai import OpenAI  # または Anthropic SDK

PROMPT_TEMPLATE = """
以下の規制情報を分析し、JSON形式で回答してください。
- summary: 200字以内の日本語要約
- impact_score: 1〜5の影響度（5が最高）
- affected_departments: 影響を受ける部門のリスト
- deadline: 施行日または対応期限（不明なら"要確認"）
- action_required: 必要なアクション（1文）

規制情報:
タイトル: {title}
ソース: {source}
本文: {body}
"""

def analyze_entry(entry):
    client = OpenAI()
    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user",
                   "content": PROMPT_TEMPLATE.format(**entry)}],
        response_format={"type": "json_object"},
    )
    return json.loads(response.choices[0].message.content)

3-2. Slack Webhook連携（コード例）

import requests

SLACK_WEBHOOKS = {
    "critical": "https://hooks.slack.com/services/T.../B.../xxx",
    "high":     "https://hooks.slack.com/services/T.../B.../yyy",
    "info":     "https://hooks.slack.com/services/T.../B.../zzz",
}

def send_alert(analysis, entry):
    score = analysis["impact_score"]
    if score >= 5:
        channel, color = "critical", "#FF0000"
    elif score >= 3:
        channel, color = "high", "#FFA500"
    else:
        channel, color = "info", "#36A64F"

    payload = {
        "attachments": [{
            "color": color,
            "title": f"[影響度{score}] {entry['source']}: {entry['title']}",
            "text": analysis["summary"],
            "fields": [
                {"title": "対応期限", "value": analysis["deadline"], "short": True},
                {"title": "影響部門", "value": ", ".join(analysis["affected_departments"]), "short": True},
            ],
            "actions": [{"type": "button", "text": "原文を見る", "url": entry["link"]}],
        }]
    }
    requests.post(SLACK_WEBHOOKS[channel], json=payload)

Microsoft Teamsの場合は、Power AutomateのWebhookコネクタに同様のJSONペイロードをPOSTしてください。

3-3. 対応ToDo管理の運用ルール

項目 ルール
起票タイミング 影響度スコア3以上のアラート受信時に自動起票
担当アサイン 影響部門のコンプライアンス担当者を自動指定
