契約トラブル予防チェックリスト

フリーランスの契約書、自力で作れますか？

契約トラブル予防チェックリスト

使い方

本チェックリストは、契約書を締結する前に使用します。自分で作成した契約書でも、クライアントから提示された契約書でも同じ手順です。

  1. 契約書を手元に用意する
  2. 各項目を上から順に確認し、該当する条項が契約書に記載されているかをYES/NOで回答する
  3. NOが1つでもあれば、その条項の追記・修正を相手方に依頼する
  4. 全項目YESになってから署名・押印する

本チェックリストの役割： 条項が「存在するか」の確認に特化しています。条項の内容が適切かの判断は、別途「契約書自動生成ツール」のインラインリスク表示を参照してください。報酬額・消費税の管理には「案件台帳」シートを併用し、税率10%（インボイス制度対応）で計算してください。

チェック項目（全10項目）

1. 業務内容・成果物の定義

確認： 委託する業務の範囲と納品する成果物が具体的に記載されているか？

  • [ ] YES　　- [ ] NO

なぜ重要か： 業務範囲が曖昧だと「これもやって」と際限なく追加作業を求められる原因になる。

こう書かれていたら危険：

「甲が別途指示する業務一切を含む」

→ 業務範囲が無限に拡大する。必ず成果物の仕様・ページ数・機能一覧など具体的な定義があることを確認する。

[記入例: YES — 第2条に「LP1ページ、レスポンシブ対応、画像10点まで」と明記あり]

2. 報酬額・支払条件

確認： 報酬金額、消費税の扱い、支払期日、支払方法が明記されているか？

  • [ ] YES　　- [ ] NO

なぜ重要か： 口約束の金額は証拠にならない。支払期日がなければ「案件台帳」の入金ステータス（O列）で管理もできない。

こう書かれていたら危険：

「報酬は甲乙協議の上決定する」

→ 金額が未確定のまま作業が始まり、完了後に値下げされるリスクがある。税率10%の消費税を含むか否かも必ず明記させる。

[記入例: YES — 第5条に「税抜300,000円、消費税10%別途、納品月末締め翌月末払い・銀行振込」と記載]

3. 納期・スケジュール

確認： 納品期限または作業期間が具体的な日付で記載されているか？

  • [ ] YES　　- [ ] NO

なぜ重要か： 納期が曖昧だと、遅延の責任の所在が不明になり、損害賠償請求の根拠にもされうる。

こう書かれていたら危険：

「納期は甲の指示に従うものとする」

→ 一方的に短納期を押し付けられる。「2026年7月31日」のように確定日付か、「発注日から30営業日以内」のように算出可能な形式であること。

[記入例: YES — 第3条に「2026年8月15日までに納品」と明記]

4. 検収条件・修正回数

確認： 成果物の検収方法、検収期間、修正対応の回数上限が記載されているか？

  • [ ] YES　　- [ ] NO

なぜ重要か： 検収条件がないと「気に入らない」という主観で永久に修正を求められ、報酬が支払われない。

こう書かれていたら危険：

「甲が満足するまで修正を行うものとする」

→ 修正回数が無限。「検収期間は納品後10営業日、修正は2回まで、以降は別途見積」のように上限を明記させる。

[記入例: NO — 検収期間の記載なし。第4条に追記を依頼する]

5. 知的財産権の帰属

確認： 成果物の著作権・知的財産権が誰に帰属するか、いつ移転するかが記載されているか？

  • [ ] YES　　- [ ] NO

なぜ重要か： 権利の帰属が不明確だと、自分のポートフォリオに実績として掲載できなくなる場合がある。

こう書かれていたら危険：

「制作過程で生じた一切の知的財産権は、創作と同時に甲に帰属する」

→ 報酬未払いでも権利だけ取られる。「報酬の完済をもって移転する」という条件付き移転にするべき。

[記入例: YES — 第8条に「報酬全額の支払完了時に甲に移転」と記載。ポートフォリオ掲載権も留保あり]

6. 秘密保持（NDA）条項

確認： 秘密情報の定義、保持義務の範囲、有効期間が記載されているか？

  • [ ] YES　　- [ ] NO

なぜ重要か： 秘密保持の範囲が広すぎると、類似案件を一切受けられなくなるリスクがある。

こう書かれていたら危険：

「業務に関連して知り得た一切の情報を秘密情報とする」

→ 公知の情報や自身が独自に持っていた知識まで秘密扱いになる。「書面で秘密と指定された情報に限る」等の限定が必要。

[記入例: YES — 第9条に秘密情報の定義あり。ただし範囲が広いため、除外規定の追記を依頼する]

7. 損害賠償の上限

確認： 損害賠償の範囲と金額の上限が記載されているか？

  • [ ] YES　　- [ ] NO

なぜ重要か： 上限がないと、報酬30万円の案件で数千万円の損害賠償を請求される理論上のリスクがある。

こう書かれていたら危険：

「乙は甲に生じた一切の損害（逸失利益を含む）を賠償する」

→ 青天井の賠償責任。「賠償額は本契約の報酬総額を上限とする」「間接損害・逸失利益は除外する」を必ず入れる。

[記入例: NO — 損害賠償条項はあるが上限の記載なし。「報酬額を上限」とする修正を依頼する]

8. 契約解除・中途終了の条件

確認： どのような場合に契約を解除できるか、解除時の報酬精算方法が記載されているか？

  • [ ] YES　　- [ ] NO

なぜ重要か： 解除条件がないと、クライアント都合で突然打ち切られても作業済み分の報酬を請求できない。

こう書かれていたら危険：

「甲は理由の如何を問わず、いつでも本契約を解除できる」

→ 受託者に一方的に不利。解除の場合も「作業済み部分の報酬は支払う」「30日前の書面通知を要する」等の条件が必要。

[記入例: YES — 第12条に双方30日前書面通知で解除可能、作業済み分は精算と記載]

9. 再委託（下請け）の可否

確認： 業務の全部または一部を第三者に再委託できるか否かが記載されているか？

  • [ ] YES　　- [ ] NO

なぜ重要か： 再委託禁止に気づかず外注を使うと契約違反になる。逆に記載がないと、どちらの解釈も成り立ち紛争の原因になる。

こう書かれていたら危険：

「乙は本業務の全部または一部を第三者に再委託してはならない」（例外規定なし）

→ 繁忙期に一切外注できない。必要に応じて「甲の事前書面承諾を得た場合は再委託可能」とする。

[記入例: YES — 第7条に「甲の書面承諾があれば再委託可能」と記載]

10. 管轄裁判所・紛争解決

確認： 紛争が生じた場合の管轄裁判所または紛争解決手段が記載されているか？

  • [ ] YES　　- [ ] NO

なぜ重要か： 管轄裁判所の記載がないと、クライアントの所在地の裁判所で訴訟になり、遠方への出廷を強いられる。

こう書かれていたら危険：

「甲の本店所在地を管轄する裁判所を第一審の専属的合意管轄裁判所とする」

→ 相手の所在地が遠方の場合、訴訟コストが大幅に増加する。自分の所在地に近い裁判所、または「被告の住所地」を管轄とする規定が望ましい。

[記入例: YES — 第15条に「東京地方裁判所」と記載。自分も東京なので問題なし]

補足：案件台帳との連携

契約締結後は「案件台帳」シートに案件を登録してください。

  • G列（報酬額） に税抜金額を入力 → H列 で消費税（H2 = G2 * 0.1）、I列 で税込合計（I2 = G2 + H2）が自動計算されます
  • M列（支払期日） を必ず入力 → O列 の入金ステータスで「⚠ 期日超過」「⏰ 期日7日以内」が自動判定されます
  • 免税事業者の場合はH列を0に変更してください

本チェックリストで条項の存在を確認 → 契約締結 → 案件台帳で報酬・入金を管理、の流れで運用すると、契約から回収までを一元管理できます。
