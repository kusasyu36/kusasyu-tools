## 2. GitHub Audit Log

**対応管理策**: A.5.18 / A.8.9 / A.8.8 / A.8.15

### 手動取得ルート

`GitHub > Organization Settings > Logs > Audit log` から期間・アクションを指定してエクスポート。

### 自動取得（GitHub CLI / REST API）

```bash
# Organization監査ログ取得（A.8.15 ログ取得）
gh api \
  -H "Accept: application/vnd.github+json" \
  "/orgs/{ORG}/audit-log?per_page=100&include=all&after=2026-01-01" \
  > github_audit_log.json

# リポジトリの権限変更記録（A.5.18 アクセス権）
gh api "/orgs/{ORG}/audit-log?per_page=100&phrase=action:org.update_member" \
  > github_permission_changes.json

# Dependabotアラート取得（A.8.8 技術的脆弱性の管理）
gh api "/repos/{OWNER}/{REPO}/dependabot/alerts?state=open" \
  > dependabot_alerts.json

# コミット履歴（A.8.9 構成管理）
gh api "/repos/{OWNER}/{REPO}/commits?since=2026-01-01T00:00:00Z&per_page=100" \
  > commit_history.json

3. Google Workspace Reports API

対応管理策: A.5.15 / A.5.17 / A.6.3 / A.8.15

手動取得ルート

Google Admin Console > レポート > 監査と調査 > 管理者ログイベント からCSVエクスポート。教育訓練記録（A.6.3）は Google Admin Console > レポート > アプリレポート > Classroom で確認。

自動取得（gcloud CLI / API）

# 管理者監査ログ（A.8.15 ログ取得）
GAM_DATE="2026-01-01T00:00:00Z"
curl -s -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  "https://admin.googleapis.com/admin/reports/v1/activity/users/all/applications/admin?startTime=${GAM_DATE}" \
  > gws_admin_audit.json

# ログインイベント・MFA状態（A.5.17 認証情報）
curl -s -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  "https://admin.googleapis.com/admin/reports/v1/activity/users/all/applications/login?startTime=${GAM_DATE}" \
  > gws_login_events.json

# ユーザー一覧・権限（A.5.15 アクセス制御）
curl -s -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  "https://admin.googleapis.com/admin/directory/v1/users?domain=example.com&maxResults=500" \
  > gws_users.json

# Driveファイルアクセスログ（A.8.15 ログ取得）
curl -s -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  "https://admin.googleapis.com/admin/reports/v1/activity/users/all/applications/drive?startTime=${GAM_DATE}" \
  > gws_drive_activity.json

補足: Google Workspaceの教育訓練受講履歴（A.6.3、担当: 山田次郎）は、eラーニングツール側のAPIから取得するケースが多い。Workspace単体ではClassroom利用ログが代替証跡となる。

4. Azure Activity Log

対応管理策: A.5.15 / A.5.18 / A.8.5 / A.8.15 / A.8.16

手動取得ルート

Azure Portal > モニター > アクティビティ ログ からフィルタ指定してCSVダウンロード。

自動取得（Azure CLI）

# アクティビティログ取得（A.8.15 ログ取得）
az monitor activity-log list \
  --start-time "2026-01-01" \
  --end-time "2026-04-11" \
  --output json > azure_activity_log.json

# ロール割り当て一覧（A.5.15 アクセス制御）
az role assignment list --all --output json > azure_role_assignments.json

# ロール変更イベント（A.5.18 アクセス権）
az monitor activity-log list \
  --start-time "2026-01-01" \
  --resource-type "Microsoft.Authorization/roleAssignments" \
  --output json > azure_role_changes.json

# サインインログ（A.8.5 セキュアな認証）
az rest --method GET \
  --url "https://graph.microsoft.com/v1.0/auditLogs/signIns?\$filter=createdDateTime ge 2026-01-01T00:00:00Z" \
  --output json > azure_signin_logs.json

# Security Centerアラート（A.8.16 監視活動）
az security alert list --output json > azure_security_alerts.json

5. Slack Audit Logs

対応管理策: A.5.18 / A.8.15

手動取得ルート

Slack Admin > セキュリティ > アクセスログ（Enterprise Grid限定）。BusinessプランではワークスペースのAnalytics画面からCSVエクスポート。

自動取得（Audit Logs API / Enterprise Grid）

# 監査ログ取得（A.8.15 ログ取得）— Enterprise Grid限定
SLACK_TOKEN="xoxp-your-token"
curl -s -H "Authorization: Bearer ${SLACK_TOKEN}" \
  "https://api.slack.com/audit/v1/logs?oldest=1735689600&limit=1000" \
  > slack_audit_logs.json

# チャンネルメンバー変更の抽出（A.5.18 アクセス権）
curl -s -H "Authorization: Bearer ${SLACK_TOKEN}" \
  "https://api.slack.com/audit/v1/logs?action=user_channel_join&oldest=1735689600" \
  > slack_channel_changes.json

# Business+プランの場合はSCIM APIでユーザー管理証跡を取得
curl -s -H "Authorization: Bearer ${SLACK_TOKEN}" \
  "https://api.slack.com/scim/v1/Users?count=200" \
  > slack_scim_users.json

注意: Audit Logs APIはEnterprise Gridプラン限定。Business+ではSCIM APIとAnalytics APIが利用可能。

6. 定期取得パイプラインの構築

方法A: cronによるローカル定期実行

# /etc/cron.d/isms-evidence（毎週月曜 9:00 に全証跡取得）
0 9 * * 1 /opt/isms/collect_all.sh >> /var/log/isms_collect.log 2>&1

collect_all.sh の構成例:

#!/bin/bash
set -euo pipefail
DATE=$(date +%Y-%m-%d)
OUT_DIR="/opt/isms/evidence/${DATE}"
mkdir -p "${OUT_DIR}"

# 各サービスの証跡取得
aws cloudtrail lookup-events --start-time "$(date -d '7 days ago' +%Y-%m-%dT00:00:00Z)" \
  --output json > "${OUT_DIR}/cloudtrail.json"
gh api "/orgs/{ORG}/audit-log?per_page=100" > "${OUT_DIR}/github_audit.json"
az monitor activity-log list --start-time "$(date -d '7 days ago' +%Y-%m-%d)" \
  --output json > "${OUT_DIR}/azure_activity.json"

# Slack通知
curl -X POST -H 'Content-type: application/json' \
  --data "{\"text\":\"✅ ISMS証跡収集完了: ${DATE}\nファイル数: $(ls ${OUT_DIR} | wc -l)件\"}" \
  "${SLACK_WEBHOOK_URL}"

方法B: GitHub Actionsによるクラウド定期実行

```yaml

.github/workflows/isms-evidence.yml

name: ISMS Evidence Collection on: schedule: - cron: '0 0 * * 1' # 毎週月曜 UTC 0:00 workflow_dispatch:

jobs: collect: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4

  - name: Configure AWS
    uses: aws-actions/configure-aws-credentials@v4
    with:
      aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
      aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
      aws-region: ap-northeast-1

  - name: Collect AWS Evidence
    run: |
      mkdir -p evidence
      aws cloudtrail lookup-events \
        --start-time "$(date -d '7 days ago' +%Y-%m-%dT00:00:00Z)" \
        --output json > evidence/cloudtrail.json
      aws iam get-credential-report --output text \
        --query 'Content' | base64 -d > evidence/credential_report.csv

  - name: Collect GitHub Audit Log
    env:
      GH_TOKEN: ${{ secrets.AUDIT_LOG_TOKEN }}
    run: |
      gh api "/orgs/${{ github.repository_owner }}/audit-log?per_page=100" \
        > evidence/github_audit.json

  - name: Upload Evidence
    uses: actions/upload-artifact@v4
    with:
      name: isms-evidence-${{ github.run_id }}
      path: evidence/
      retention-days: 365

  - name: Slack Notification
    env:
      SLACK_WEBHOOK: ${{ secrets.SLACK_WEBHOOK_URL }}
    run: |
      COUNT=$(ls evidence/ | wc -l)
      curl -X POST -H 'Content-type: application/json' \
        --data "{\"text\":\"✅ ISMS証跡を${COUNT}件収集しました（Run: ${{ github.run_id }}）\"}" \
        "${SLACK_WEBHOOK}"
