法務precedentキャプチャ台帳

PrecedentDB — 法務判断precedent即時検索エンジン＆導入ROIシミュレーター

法務Precedentキャプチャ台帳 — Excel対応テンプレート構築ガイド

概要

法務部門の判断履歴を構造化し、属人化を防ぎ、過去の判例を即座に参照可能にするExcel台帳テンプレート。数式・条件付き書式をプリセットし、入力するだけで分析が自動化される。

1. シート構成

シート名 用途
台帳 メインの入力シート（1レコード＝1判断）
マスタ 契約種別・条項タイプのマスタリスト
サマリー カバレッジ率・HHI・グラフ集約

2. 台帳シート — 列定義

入力列（A〜J列）

列 ヘッダー 入力形式 備考
A 通番 整数 連番。=ROW()-1 で自動採番可
B 日付 yyyy/mm/dd 判断実施日
C 契約種別 ドロップダウン マスタシートから参照
D 条項タイプ ドロップダウン マスタシートから参照
E 相手方要求概要 テキスト 200字以内推奨
F 自社判断 ドロップダウン 承認／条件付承認／拒否／保留
G 判断理由 テキスト 根拠を簡潔に記録
H 判断者 テキスト 氏名またはイニシャル
I 参照Precedent番号 整数 過去の通番を入力（なければ空欄）
J ステータス ドロップダウン 有効／更新済／廃止

自動計算列（K〜M列）

列 ヘッダー 数式 説明
K 経過日数 =IF(B2="","",TODAY()-B2) 判断からの日数
L 要更新フラグ =IF(K2="","",IF(K2>90,"要更新","")) 90日超で自動表示
M 被参照回数 =COUNTIF(I:I,A2) この判断が参照された回数

3. マスタシート — 設定内容

契約種別マスタ（A列）

業務委託契約 / 秘密保持契約(NDA) / ライセンス契約 / 売買契約
利用規約 / 共同研究契約 / 雇用契約 / 代理店契約 / その他

条項タイプマスタ（B列）

損害賠償 / 責任制限 / 秘密保持 / 競業避止 / 知的財産
契約解除 / 準拠法・管轄 / 保証条項 / 不可抗力 / 反社排除
個人情報保護 / 権利帰属 / 支払条件 / 契約期間 / その他

条項タイプ数は =COUNTA(マスタ!B:B)-1 で自動カウント（サマリーシートで使用）。

4. サマリーシート — 自動集計

4-1. 条項タイプ別カバレッジ率

セル 内容 数式例
A列 条項タイプ名 =マスタ!B2 で一覧をミラー
B列 記録件数 =COUNTIF(台帳!D:D,A2)
C列 カバレッジ率 =IF(COUNTA(B:B)-1=0,0,COUNTIF(B:B,">"&0)/（条項タイプ総数）)

カバレッジ率の定義: 1件以上の判断記録がある条項タイプ数 ÷ 全条項タイプ数。100%に近いほど網羅的。

グラフ: B列を横棒グラフで可視化。カバレッジ率C列は単一セルにパーセント表示。

4-2. 判断者集中度 HHI（ハーフィンダール指数）

セル 内容 数式例
E列 判断者名 ユニーク抽出（手動 or UNIQUE関数）
F列 件数 =COUNTIF(台帳!H:H,E2)
G列 シェア =F2/SUM(F:F)
H列 シェア二乗 =G2^2
HHI合計 =SUM(H:H)

HHI判定基準:

HHI値 判定 対応
0.00〜0.25 分散（健全） 現状維持
0.25〜0.50 やや集中 クロストレーニング推奨
0.50〜1.00 高度集中（属人化） 即時是正が必要

グラフ: 判断者別件数を円グラフ、HHI値をゲージ表示で可視化。

4-3. 鮮度分布

セル 内容 数式例
J2 有効レコード数 =COUNTIF(台帳!J:J,"有効")
J3 うち90日超 =COUNTIFS(台帳!J:J,"有効",台帳!K:K,">"&90)
J4 要更新率 =IF(J2=0,0,J3/J2)

5. 条件付き書式の設定

対象範囲 条件 書式
台帳シート 各行全体 =$K2>90 行背景を黄色（#FFFF00）
L列「要更新フラグ」 セル値＝"要更新" 赤太字
F列「自社判断」＝拒否 セル値＝"拒否" 赤背景・白文字
M列 被参照回数≧3 セル値≧3 緑太字（重要判例を視認）

6. ドロップダウンリストの設定手順

  1. マスタシートを名前定義: 契約種別 → =マスタ!$A$2:$A$10、条項タイプ → =マスタ!$B$2:$B$16
  2. 台帳シートで適用: C列全体 → データの入力規則 → リスト → =契約種別
  3. D列・F列・J列も同様に設定

7. CSV出力の手順と注意点

出力手順

  1. 台帳シートを選択
  2. 「ファイル」→「名前を付けて保存」→「CSV UTF-8（コンマ区切り）」
  3. ファイル名例: precedent_export_20260227.csv

CSV互換性のための注意

  • テキスト列（E・G列）にカンマが含まれる場合、Excelが自動でダブルクォート囲みを付与
  • 日付列はyyyy/mm/dd形式で出力されるため、インポート先のパーサーに合わせて変換
  • 数式列（K〜M）は計算結果の値としてエクスポートされる
  • BOM付きUTF-8で出力すれば日本語文字化けを防止

8. 運用ルール（推奨）

ルール 内容
記録タイミング 判断の当日中に入力
月次レビュー サマリーシートで要更新率・HHIを確認
四半期棚卸 ステータス「有効」の90日超レコードを精査し、更新 or 廃止
参照Precedent 類似案件は必ず過去番号を入力し、判断の一貫性を担保
バックアップ 月初にファイルコピーを保存（ファイル名に日付付与）
