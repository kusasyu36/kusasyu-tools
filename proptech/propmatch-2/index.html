1. 住所表記ルール

1-1. 基本フォーマット

{都道府県}{市区町村}{町名}{丁目}-{番地}-{号} [{建物名} {部屋番号}]

例: 東京都渋谷区恵比寿1-5-12 恵比寿ガーデンプレイス301

1-2. 区切り文字・表記ルール

項目 ルール OK例 NG例
丁目〜番地〜号 ハイフン - 統一（全角・漢字禁止） 1-5-12 1丁目5番12号 / １−５−１２
都道府県 省略不可（DB登録時は必須） 東京都 東京
市区町村 正式名称。略称・通称禁止 渋谷区 渋谷
建物名 スペース1つで区切り 恵比寿G 301 恵比寿G-301
部屋番号 半角数字。階層は F で表記 203 / 3F-A ２０３
番地のみ（号なし） 番地のみ記載 1-5 1-5-

1-3. 都道府県省略ルール

  • DB・システム登録: 省略不可（バリデーション必須）
  • 帳票・チラシ出力: 同都道府県内のみ省略可（設定で制御）
  • 外部連携・API送信: 常に省略不可

2. 物件ID付番規則

2-1. ID体系

[エリアコード2桁]-[種別コード1桁]-[西暦下2桁][連番5桁]-[枝番2桁]

例: 13-R-2412345-01

2-2. 各フィールド定義

フィールド 桁数 内容 例
エリアコード 2桁 都道府県コード（JIS X 0401） 13=東京、27=大阪
種別コード 1桁 R=賃貸, S=売買, C=商業, L=土地 R
西暦下2桁 2桁 登録年 24=2024年
連番 5桁 000001〜99999（ゼロパディング） 00345
枝番 2桁 部屋・区画の枝番（親物件は01） 01

2-3. 枝番ルール

  • マンション1棟: 親ID（枝番01）＋各部屋（枝番02〜）
  • 戸建て: 枝番01固定
  • 区画分譲: 区画ごとに枝番付与（最大99）
  • 再登録禁止: 廃番IDは欠番処理。再利用不可

HTMLツール判定ロジックとの紐づけ

本対照表は住所名寄せHTMLツールと以下のパラメータで連動します：

距離閾値（編集距離）: ≤2 → 自動補正候補として提示
距離閾値: 3〜5 → 要確認フラグ（黄）
距離閾値: ≥6 → 手動修正必要（赤）

パターン判定優先順位:
  1. 正規表現マッチ（全角/漢字区切り）→ 即時変換
  2. 編集距離計算 → 閾値判定
  3. 住所マスタDB照合 → 実在確認

5. システム実装ガイド

5-1. Google Sheets（推奨スタート環境）

セットアップ手順:

  1. スプレッドシート新規作成 → シート名を 物件マスタ に変更
  2. A1〜J1行にヘッダー設定（物件ID、都道府県、市区町村、町名、番地、建物名、部屋番号、賃料、面積、築年月）
  3. データ検証を設定:
  4. 物件ID列: =REGEXMATCH(A2,"^\d{2}-[RSCL]-\d{7}-\d{2}$")
  5. 番地列: =REGEXMATCH(E2,"^\d+-\d+(-\d+)?$")
  6. 条件付き書式で違反セルを赤ハイライト
  7. Apps Scriptで重複IDチェック関数を追加
// Apps Script: 重複ID検出
function checkDuplicateID() {
  const sheet = SpreadsheetApp.getActiveSheet();
  const ids = sheet.getRange('A2:A').getValues().flat().filter(v => v);
  const dupes = ids.filter((id, i) => ids.indexOf(id) !== i);
  if (dupes.length > 0) {
    SpreadsheetApp.getUi().alert('重複ID: ' + dupes.join(', '));
  }
}

5-2. Redmine（プロジェクト管理連携）

カスタムフィールド設定:

  1. 管理 → カスタムフィールド → 「チケット」で新規作成
  2. 以下フィールドを追加:
フィールド名 形式 正規表現
物件ID テキスト ^\d{2}-[RSCL]-\d{7}-\d{2}$
住所（正規化済） テキスト 必須・都道府県含む
チェックリスト完了 チェックボックス10個 —
  1. ワークフロー設定: チェックリスト未完了時は「登録完了」ステータスへ遷移不可
  2. Webhook設定でHTMLツールと連携（住所バリデーションAPI呼び出し）

5-3. Oracle Database（本番DB環境）

テーブル定義 + 制約:

CREATE TABLE BUKKEN_MASTER (
  BUKKEN_ID    VARCHAR2(15)  NOT NULL,
  PREF         VARCHAR2(10)  NOT NULL,
  CITY         VARCHAR2(20)  NOT NULL,
  TOWN         VARCHAR2(30)  NOT NULL,
  BANCHI       VARCHAR2(15)  NOT NULL,
  BLDG_NAME    VARCHAR2(50),
  ROOM_NO      VARCHAR2(10),
  RENT         NUMBER(10),
  AREA_M2      NUMBER(7,2),
  BUILT_YM     CHAR(7),
  CONSTRAINT PK_BUKKEN PRIMARY KEY (BUKKEN_ID),
  CONSTRAINT CK_BUKKEN_ID CHECK (
    REGEXP_LIKE(BUKKEN_ID, '^\d{2}-[RSCL]-\d{7}-\d{2}$')
  ),
  CONSTRAINT CK_BANCHI CHECK (
    REGEXP_LIKE(BANCHI, '^\d+-\d+(-\d+)?$')
  ),
  CONSTRAINT CK_BUILT_YM CHECK (
    REGEXP_LIKE(BUILT_YM, '^\d{4}-\d{2}$')
  )
);

バリデーションプロシージャ:

-- 住所の全角チェック（揺れ検出）
CREATE OR REPLACE PROCEDURE validate_address(p_id IN VARCHAR2) AS
  v_banchi VARCHAR2(15);
BEGIN
  SELECT BANCHI INTO v_banchi FROM BUKKEN_MASTER WHERE BUKKEN_ID = p_id;
  IF REGEXP_LIKE(v_banchi, '[０-９－]') THEN
    RAISE_APPLICATION_ERROR(-20001, '全角文字検出: ' || p_id);
  END IF;
END;

5-4. SAP（大規模不動産管理ERP）

マスタデータ設定手順:

  1. トランザクション MM01 → 「不動産マスタ」設定
  2. 命名規約をカスタムフィールド ZZPROPID に登録
  3. フィールド長: 15桁
  4. チェックルーティン: Z_CHECK_PROP_ID（ABAPで実装）
  5. ABAPバリデーション関数:
FUNCTION Z_CHECK_PROP_ID.
  DATA: lv_regex TYPE string VALUE '^\d{2}-[RSCL]-\d{7}-\d{2}$'.
  IF NOT im_propid CP lv_regex.
    MESSAGE 'ID形式エラー' TYPE 'E'.
  ENDIF.
ENDFUNCTION.
  1. IDOCインターフェース設定: 外部連携時も同規約を強制（セグメント E1PROPID に制約追加）
