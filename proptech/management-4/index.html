物件データ統一管理マスターシート

物件入力は1回だけ。ポータル一括変換ツール

物件データ統一管理マスターシート — 4ポータル一括入力ガイド

なぜ「統一管理シート」が必要なのか

不動産仲介の現場では、1物件につきREINS・SUUMO・アットホーム・HOMES（HOME'S）の4ポータルへ個別にデータを入力しています。各ポータルの必須項目は微妙に異なり、入力漏れや数値の不一致が頻発します。

よくあるトラブル：

  • SUUMOには駐車場情報を入れたのにアットホームでは空欄だった
  • REINSの専有面積と他ポータルの数値が0.01㎡ズレている
  • HOMESだけ設備欄が未入力のまま公開されていた

本シートは「1行1物件・全ポータルの必須項目を横並び」で管理し、入力完了率を自動算出することでこれらのミスを防ぎます。

列構成と色分けルール（全78列）

78列を5つのカテゴリに分類し、ポータルごとにセル背景色を設定します。

色 ポータル カラーコード
🔵 青 REINS #D6E4F0
🟢 緑 SUUMO #D9EAD3
🟠 橙 アットホーム #FCE5CD
🔴 赤 HOMES #F4CCCC
⚪ 白 共通（全ポータル共通項目） #FFFFFF

共通項目は白背景とし、特定ポータルだけで必要な項目にのみ色を付けます。これにより「この列はどのポータル用か」が一目で判別できます。

入力完了率の数式

各ポータルの必須列を対象にCOUNTBLANKで空欄数をカウントし、完了率をパーセント表示します。

REINS完了率（BT列）の例

=1 - COUNTBLANK(F2,G2,I2,J2,K2,L2,N2,P2,Q2,S2,T2,U2,AB2,AC2,AD2,AE2,AF2,AG2,AI2,AJ2,AK2,AM2,AN2,AO2,AP2,AR2,AS2,AT2,AU2,BJ2,BP2)
  / 31

REINSの必須項目31列のうち、空欄でない割合を算出します。

SUUMO完了率（BU列）の例

=1 - COUNTBLANK(F2,G2,H2,I2,J2,K2,L2,M2,N2,P2,Q2,R2,S2,T2,U2,Y2,Z2,AA2,AD2,AE2,AH2,AK2,AL2,AM2,AN2,AO2,AQ2,AR2,AS2,AV2,AW2,AX2,AZ2,BA2,BB2,BC2,BD2,BE2,BF2,BH2,BI2,BK2,BM2,BQ2)
  / 44

全体完了率（BX列）

=1 - COUNTBLANK(F2:BS2) / 66

管理用5列（A〜E）と完了率列（BT〜BX）を除く66列が対象です。

80%〜99% 黄 #FFF2CC あと少し
80%未満 赤 #F4CCCC 要対応

入力ミスを減らし、CSVエクスポート時の表記ゆれも防止できます。

CSVエクスポート → HTMLチェッカー連携手順

ステップ1：対象物件をフィルタリング

物件マスターシートで「ステータス = 公開中」かつ対象ポータルの完了率が100%の行だけを表示します。

ステップ2：CSV形式で保存

ファイル → 名前を付けて保存 → ファイルの種類「CSV UTF-8（コンマ区切り）」

注意点： - 文字コードは必ず UTF-8 を選択（Shift-JISだと文字化けする） - フィルタ表示中に保存すると、表示行のみがエクスポートされる（Excel仕様）

ステップ3：HTMLチェッカーに読み込む

1. ブラウザでHTMLチェッカーを開く
2. 「CSVファイルを選択」ボタンをクリック
3. 保存した .csv ファイルを選択
4. チェック結果が画面に表示される

チェッカーは以下を自動検証します：

  • 必須項目の空欄チェック（ポータル別）
  • 数値範囲チェック（価格が0以下、面積が負数でないか等）
  • 文字数チェック（キャッチコピー・コメントの上限超過）
  • 整合性チェック（マンションなのに土地面積が入力されている等）

ステップ4：エラー修正と再チェック

チェック結果でエラーが出た場合は、Excelに戻って該当セルを修正し、再度CSV保存 → チェッカー読み込みを繰り返します。

運用のコツ

1. 入力順序を決める

共通項目（白列）→ REINS（青列）→ SUUMO（緑列）→ アットホーム（橙列）→ HOMES（赤列）

共通項目を先に埋めると、ポータル固有列だけに集中できます。

2. 完了率でフィルタする習慣

毎朝「全体完了率80%未満」でフィルタをかけ、未入力物件を洗い出すルーティンを推奨します。

3. 写真管理との連携

BL列（写真フォルダパス）にはネットワークドライブのパスを入力します。

例：\\server\photos\P-20260401-001\

写真枚数（BK列）と実際のフォルダ内ファイル数を突き合わせることで、写真のアップロード漏れも防止できます。

4. 複数担当者での共同編集

SharePointやGoogleドライブに配置して共同編集する場合、担当者列（D列）でフィルタすれば自分の物件だけ表示できます。完了率が自動更新されるため、進捗管理にも使えます。

まとめ

課題 本シートでの解決策
ポータルごとに別ファイルで管理 1シートに78列で集約
入力漏れに気づかず公開 COUNTBLANK数式で完了率を自動表示
どの列がどのポータル用か不明 背景色4色で視覚的に判別
入力値の表記ゆれ ドロップダウンで選択肢を統一
公開前の最終チェックが手作業 CSVエクスポート→HTMLチェッカーで自動検証

4ポータル×数十物件の管理が1つのExcelファイルで完結します。まずは既存物件5件でテスト入力し、完了率100%を達成できるか試してみてください。
