BtoB企業向けAIコンテンツ量産パッケージ サービス設計書

文書バージョン: 1.0 作成日: 2026年2月21日 作成者: SynLab Digital Marketing 機密区分: 社外秘

1. サービス概要

1-1. サービスコンセプト

「ContentForge（コンテンツフォージ）」は、BtoB中小企業のマーケティング部門が抱える「コンテンツ制作のリソース不足」と「専門性の壁」を同時に解決するサブスクリプション型コンテンツ量産サービスである。

GA4のアクセスデータとGoogle Search Consoleの検索クエリを分析し、AIが業界特化型のSEO記事・ホワイトペーパー・メルマガ原稿を半自動で生成する。すべてのコンテンツは専任編集者による監修を経て納品されるため、「AIっぽさ」を排除した実用品質を担保する。

1-2. 解決する課題

課題 ContentForgeの解決策
社内にライターがいない AI + 専任編集者が代行
業界専門知識が必要で外注しにくい 業界特化プロンプト + 用語辞書を構築
何を書けばいいかわからない GA4・検索クエリから自動でテーマ提案
制作に時間がかかりすぎる AI下書きで制作期間を1/3に短縮
SEO効果が見えない 月次レポートでPV・順位・CVを可視化

1-3. ターゲット顧客像

  • 企業規模: 従業員30〜300名のBtoB企業
  • 業種: 製造業、SaaS、専門サービス業（コンサル・士業・人材）
  • 担当者: マーケティング部門の責任者、または経営者が兼務
  • 現状: 営業主体の集客に限界を感じ、Webマーケティングへ移行を検討中
  • 予算感: 月額10〜50万円のマーケティング投資を想定

1-4. 競合優位性

  1. データドリブンなテーマ選定: GA4連携により「勘」ではなく「数字」でコンテンツを決める
  2. 業界特化の品質: 汎用AI出力ではなく、業界用語辞書・過去コンテンツ学習による専門性
  3. 半自動ワークフロー: 完全AI任せでも完全人力でもない、最適なコスト・品質バランス
  4. 透明な効果測定: 納品して終わりではなく、コンテンツごとのパフォーマンスを月次で報告

3. ワークフロー

3-1. コンテンツ制作の全体フロー

[STEP 1]        [STEP 2]       [STEP 3]       [STEP 4]       [STEP 5]
データ分析  →  テーマ承認  →  AI下書き生成 →  人間監修・編集 →  納品・入稿
(自動/2日)    (顧客/3日)    (自動/1日)     (編集者/3日)    (担当者/1日)
                                                    ↓
                                              [STEP 5.5]
                                              顧客確認・修正
                                              (顧客/3日+編集/2日)

標準リードタイム: テーマ承認から最終納品まで10営業日

3-2. 各ステップの詳細

STEP 1｜データ分析（自動・2営業日）

  • GA4から過去90日間のページ別PV・滞在時間・離脱率を自動取得
  • Google Search Consoleから検索クエリ・表示回数・CTR・平均掲載順位を取得
  • 競合サイトの上位表示キーワードをクロール分析
  • 上記データを統合し、「検索ボリューム × 競合難易度 × 自社関連度」でスコアリング
  • スコア上位からテーマ候補を月間制作本数の1.5倍リストアップ

STEP 2｜テーマ承認（顧客・3営業日）

  • テーマ候補リストをSlackまたはメールで送付
  • 各テーマに「推定検索ボリューム」「競合難易度」「推奨記事構成案」を添付
  • 顧客がテーマを選択・優先順位を指定
  • 3営業日以内に返答がない場合、スコア上位から自動選定（事前合意の上）

STEP 3｜AI下書き生成（自動・1営業日）

  • 承認テーマに基づき、業界特化プロンプトでAIが下書きを生成
  • SEO記事: 見出し構成（H2/H3）→ 本文 → メタディスクリプション → 内部リンク提案
  • ホワイトペーパー: 章立て → 各章本文 → 図表指示書 → CTAテキスト
  • メルマガ: 件名候補3案 → 本文 → CTA → A/Bテスト案
  • 業界用語辞書・過去コンテンツ・トンマナガイドを参照し、一貫性を担保

STEP 4｜人間監修・編集（編集者・3営業日）

  • 専任編集者がAI下書きを全文チェック
  • ファクトチェック（数値・固有名詞・法的表現の正確性確認）
  • 業界専門家としての視点追加（独自見解・事例・具体的アドバイス）
  • SEO最適化の最終調整（キーワード密度・見出し最適化・構造化データ）
  • トンマナ・表記ルールの統一
  • 著作権・薬機法等のリーガルチェック

STEP 5｜納品・入稿（担当者・1営業日）

  • Google Docs or Markdownで納品（Growthプラン以上はCMS直接入稿）
  • アイキャッチ画像の選定・設置（商用フリー素材 or Canvaテンプレート）
  • メタディスクリプション・OGP設定
  • 顧客による最終確認 → 修正対応 → 公開

3-3. 品質管理チェックリスト

各コンテンツは納品前に以下を全項目クリアする。

  • [ ] SEOターゲットKWが本文・見出しに自然に含まれている
  • [ ] ファクトチェック完了（数値の出典明記、固有名詞の正確性）
  • [ ] コピー率チェック5%以下（CopyContentDetectorで検証）
  • [ ] 読みやすさスコアが基準値以上（文章の平均文長40字以内）
  • [ ] CTA（行動喚起）が適切に配置されている
  • [ ] 画像のalt属性・ファイル名がSEO最適化されている
  • [ ] 社内表記ルール・トンマナガイドに準拠している

5-5. 免責事項

  • 天災・パンデミック等の不可抗力事象
  • 顧客の事業方針変更に伴う大幅な仕様変更要請

6-2. 主要コンポーネント

データ分析モジュール

  • GA4 Data API v2によるアクセスデータの自動取得（日次バッチ）
  • Google Search Console APIによる検索パフォーマンスデータ取得
  • 競合分析にはDataForSEO APIを利用（キーワード難易度・SERP分析）
  • 分析結果はPostgreSQLに蓄積し、時系列でトレンドを追跡

AI生成エンジン

  • メインLLM: Claude API（Sonnet 4.6 / Opus 4.6を用途別に使い分け）
  • 構成生成（見出し設計）: Sonnet 4.6（高速・低コスト）
  • 本文生成: Opus 4.6（高品質・長文対応）
  • プロンプトテンプレートは業界別にバージョン管理（Git）
  • 業界用語辞書・トンマナガイドをRAG（検索拡張生成）で参照

品質チェックエンジン

  • コピー率チェック: CopyContentDetector API連携
  • 読みやすさスコア: textstat相当のカスタムロジック
  • SEOチェック: ターゲットKW含有率・見出し構造・メタ情報の自動検証
  • 最終的な品質判定は人間の編集者が実施（AIは補助）

ワークフローエンジン

  • n8n（セルフホスト）でデータ取得〜テーマ提案〜AI生成〜品質チェックを自動化
  • 人間の承認・編集が必要なステップはSlack通知 + ダッシュボード上で対応
  • 各ステップのステータス・所要時間を自動記録し、ボトルネックを可視化

6-3. セキュリティ & データ管理

項目 対応内容
顧客データの取り扱い GA4/GSCデータは当社クラウド環境内でのみ処理。第三者提供なし
API認証情報 AWS Secrets Manager で暗号化管理
通信暗号化 全通信をTLS 1.3で暗号化
アクセス制御 担当者ごとにIAMロールを設定、最小権限の原則
データ保持期間 契約終了後90日で全データを削除（顧客要望で即時削除可）
監査ログ 全操作ログをCloudWatch Logsに1年間保持
AI学習への利用 顧客コンテンツをAIモデルの学習データに使用しない（API利用のみ）

6-4. インフラ構成

  • クラウド: AWS（東京リージョン）
  • コンピュート: ECS Fargate（コンテナベース、サーバーレス）
  • データベース: Amazon RDS（PostgreSQL 16）
  • ファイルストレージ: Amazon S3（納品物・画像素材の保管）
  • ワークフロー: n8n on ECS（セルフホスト）
  • モニタリング: CloudWatch + PagerDuty（障害時の自動アラート）
