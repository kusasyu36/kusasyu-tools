このガイドの位置づけ

本ガイドは「行動トリガー設計ツール」で生成したシナリオを、実際のポップアップツールに実装するための手順書である。

役割 担当
シナリオ設計（トリガー条件・CTA文言・表示タイミング） 行動トリガー設計ツール
実装手順（ツールの設定画面でどこに何を入力するか） 本ガイド

設計ツールで生成済みのシナリオID（TRG-H001、TRG-C001、TRG-E001など）をそのまま各ツールのキャンペーン名に使用することで、効果測定CSVとの突合が容易になる。

対応する無料・低価格ツール

ツール名 無料枠 特徴
Privy 月5,000PVまで無料 カート放棄トリガーが無料枠で使える
OptiMonk 月15,000PVまで無料 スクロール・滞在時間トリガー対応
Promolayer 月10,000表示まで無料 日本語UI・国産ツール

場面1：カート放棄（TRG-H001）

なぜこのタイミングが機能するか

カートに商品を入れた訪問者は「購買意図の表明」を済ませた状態にある。行動経済学の「保有効果（Endowment Effect）」により、カートに入れた時点で心理的に"自分のもの"と認知し始める。離脱しようとする瞬間に介入すると、「せっかく選んだのに手放すのか」という損失回避バイアスが働き、完了率が上がる。TRG-H001（カート放棄_購入催促ポップアップ）のA/Bテストでは、表示回数1,250回に対しCV数12件（平均注文単価4,980円・健康食品カテゴリ）という結果が出ており、介入なしとの差が明確に現れるトリガーである。

ツール別の設定手順

Privy の場合

  1. ダッシュボード → Create New Campaign → 「Exit Intent」を選択
  2. Trigger Settings で以下を入力
  3. Trigger Type：Exit Intent
  4. Show on pages：/cart（カートページURL）
  5. Frequency：Once per session
  6. Design タブで設計ツールが生成したCTA文言をそのまま貼り付け
  7. Campaign Name に TRG-H001_A または TRG-H001_B と入力（テストパターン識別用）

OptiMonk の場合

  1. Campaigns → New Campaign → テンプレート「Cart Abandonment」選択
  2. Who to show → Page targeting → Contains /cart
  3. When to show → Trigger：On exit intent を選択
  4. A/Bテスト設定：Add Variant でパターンBを追加し、設計ツールの各パターン文言を入力

Promolayer の場合

  1. 新規キャンペーン → ポップアップ種別「離脱防止」
  2. 表示条件 → ページURL /cart を含む → 離脱検知ON
  3. テキスト欄に設計ツールの生成文言を入力
  4. キャンペーン名に TRG-H001 を設定

場面2：長時間滞在／スクロール深度（TRG-C001）

なぜこのタイミングが機能するか

商品ページでスクロール深度70%以上に達したユーザーは、価格・成分・仕様といった詳細情報まで読み込んでおり、「情報収集フェーズ」から「比較検討フェーズ」に移行している。社会的証明（Social Proof）理論によると、この段階で第三者のレビューや購入実績を提示すると「自分だけでなく他者も選んでいる」という安心感が意思決定の後押しになる。TRG-C001（スクロール70%_レビュー訴求）では、コスメカテゴリで表示回数2,100回・CV数22件（平均注文単価3,800円）とCTRが高く、情報を深く読んだユーザーほどレビュー訴求への反応が良いことが確認されている。

ツール別の設定手順

Privy の場合

  1. Create New Campaign → 「Scroll Box」タイプを選択
  2. Trigger Settings
  3. Trigger Type：Scroll percentage
  4. Scroll depth：70%
  5. Show on pages：商品詳細ページのURL群（/products/*）
  6. テキストに設計ツールのレビュー訴求メッセージを貼り付け
  7. Campaign Name：TRG-C001_A

OptiMonk の場合

  1. New Campaign → 「Nano bar」または「Popup」を選択
  2. When to show → Trigger：After scrolling down → 値を 70% に設定
  3. Who to show → Page targeting：/products/ を含むURL
  4. 表示頻度：Maximum 1 per sessionに設定（過剰表示を防止）

Promolayer の場合

  1. 新規キャンペーン → 表示トリガー「スクロール率」
  2. スクロール率を 70% に設定
  3. 表示ページ：商品詳細ページを指定
  4. キャンペーン名：TRG-C001

補足：滞在時間トリガー（例：60秒以上）も同じ心理的根拠で機能する。OptiMonkでは When to show → After X seconds で秒数指定が可能。スクロール深度と併用する場合は「OR条件」で設定し、いずれか早い方で発火させる。

場面3：比較閲覧（複数商品ページ遷移）

なぜこのタイミングが機能するか

複数の商品ページを行き来するユーザーは「選択のパラドックス」に陥りやすい。選択肢が多いほど意思決定が困難になり、結果として「何も買わない」を選んでしまう。この段階で「人気No.1」「この商品が一番選ばれています」といったデフォルト選択肢を提示すると、ヒューリスティック（簡便な判断規則）として機能し、意思決定コストが下がる。設計ツールで生成した比較閲覧シナリオのCTA文言は、この心理的負荷の軽減を狙って設計されている。

ツール別の設定手順

OptiMonk の場合（ページビュー数トリガー対応）

  1. New Campaign → Popup テンプレートを選択
  2. Who to show → Visitor behavior → Number of visited pages → 3 pages or more に設定
  3. Page targeting：商品カテゴリページ（/products/ 配下）
  4. 設計ツールの比較閲覧用CTA文言（例：「迷ったらこちら！今月の人気ランキング」）を入力

Privy の場合

Privyには直接的なページビュー数トリガーがないため、以下の代替設定を使用する。

  1. Create New Campaign → 「Bar」タイプ選択
  2. Trigger：Timer → 15 seconds（複数ページ閲覧に要する平均時間）
  3. Show on pages：カテゴリ一覧ページのURLを指定
  4. この設定は近似値であり、正確なページ遷移回数による制御にはOptiMonkを推奨

Promolayer の場合

  1. 新規キャンペーン → 表示条件「訪問ページ数」
  2. ページ数を 3 以上に設定
  3. 設計ツールの生成文言を入力

場面4：再訪問／初回訪問（TRG-E001）

なぜこのタイミングが機能するか

再訪問ユーザーは「単純接触効果（Mere Exposure Effect）」により、初回訪問時よりもブランドへの好意度が上がっている。さらに、前回の閲覧履歴をリマインドすることで「中断した行動の完了欲求（ツァイガルニク効果）」が発動する。TRG-E001（再訪問_閲覧履歴リマインド）では、家電カテゴリで表示回数520回・CV数6件（平均注文単価15,800円）と、高単価商材で特に効果が大きい。一方、初回訪問者には「ウェルカムオファー」として初回限定特典を提示し、損失回避（今だけの機会を逃す恐怖）で初回購入のハードルを下げる。

ツール別の設定手順

再訪問トリガー（TRG-E001対応）

ツール 設定場所 入力値
OptiMonk Who to show → Visitor type Returning visitors を選択
Privy Trigger → Audience Has visited before にチェック
Promolayer 表示条件 → 訪問回数 2回目以降 を選択

いずれのツールでも、キャンペーン名に TRG-E001 を入力し、設計ツールで生成した「前回ご覧いただいた商品はまだ在庫がございます」等のリマインド文言をそのまま使用する。

初回訪問トリガー

ツール 設定場所 入力値
OptiMonk Who to show → Visitor type New visitors を選択
Privy Trigger → Audience First-time visitors にチェック
Promolayer 表示条件 → 訪問回数 初回 を選択

初回訪問用のシナリオは設計ツールで「初回訪問」トリガー種別を選択して生成し、ウェルカムクーポンや送料無料オファーの文言をここに入力する。

実装チェックリスト

すべてのシナリオを実装する際に、以下を確認する。

# チェック項目 理由
1 キャンペーン名にシナリオID（TRG-H001等）を含めたか 効果測定CSVとの突合に必須
2 A/Bパターンを別キャンペーンとして作成したか パターンA・Bの比較検証のため
3 表示頻度を「1セッション1回」に制限したか 過剰表示はUX悪化と離脱率上昇を招く
4 モバイル表示を確認したか EC訪問の70%以上がスマートフォン経由
5 CTA文言は設計ツールの出力をそのまま使用したか 独自に改変するとA/Bテストの統制が崩れる

効果測定との連携

実装後の効果測定は、設計ツールが出力するCSVテンプレート（計測週・シナリオID・表示回数・CTR・CVR・売上貢献額・ROI等を含む）に各ツールのレポート数値を転記する。例えばTRG-H001のパターンAであれば、Privyのダッシュボードから表示回数・クリック数・CV数を取得し、CSVの該当行に入力することで、前週CVRとの変化率や次アクション判定（継続／改善／停止）が自動算出される。

まとめ

行動トリガーの効果は「シナリオの質 × 実装の正確さ」で決まる。設計ツールで心理学的根拠に基づいたシナリオを生成し、本ガイドの手順で無料ツールに正しく実装すれば、月商500万〜5,000万円規模のEC運営でも大手が使う自動化ロジックと同等の介入施策を1〜3名体制で運用できる。まずはカート放棄（TRG-H001）から実装し、効果を確認した上でスクロール深度（TRG-C001）、再訪問（TRG-E001）へと展開することを推奨する。
