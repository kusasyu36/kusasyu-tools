1. テンプレートアーキテクチャ概要

1.1 共通構造

すべての業種テンプレートは以下の3層構造で設計する。

┌─────────────────────────────────┐
│  Layer 1: 業種マスター定義       │  ← 業種固有の語彙・トーン・制約
├─────────────────────────────────┤
│  Layer 2: コンテンツタイプ定義    │  ← ブログ / SNS / メルマガの型
├─────────────────────────────────┤
│  Layer 3: バリエーション展開ルール │  ← 季節・ペルソナ・目的別の分岐
└─────────────────────────────────┘

1.2 共通入力項目（全業種共通）

入力項目 型 必須 説明
company_name string Yes 会社名・屋号
industry enum Yes 業種選択（5業種から）
product_service string Yes 主力商材・サービス名（最大3つ）
target_customer string Yes 想定顧客像（自由記述 or 選択式）
tone enum No 文体指定（デフォルト: 業種推奨値）
region string No 商圏エリア（地域密着型の場合）
keywords string[] No 狙いたいSEOキーワード（最大5つ）

1.3 共通出力形式

コンテンツタイプ 出力フォーマット 標準文字数
SEOブログ記事 Markdown / HTML 3,000〜5,000字
SNS投稿（X） プレーンテキスト 〜140字
SNS投稿（Instagram） テキスト＋ハッシュタグ 〜300字＋タグ30個以内
メルマガ HTML（MJML互換） 800〜1,500字

2. 製造業テンプレート

2.1 業種マスター定義

定義項目 値
業種コード MFG
推奨トーン professional（信頼性重視・技術的正確さ）
一人称 「当社」「弊社」
禁止表現 誇大広告表現、未検証の数値、競合社名の直接言及
E-E-A-T強化要素 技術者の実名コメント枠、ISO認証・特許番号の挿入欄、導入事例の定量データ

2.2 業種固有入力項目

入力項目 型 必須 説明
manufacturing_type enum Yes 加工種別（切削/プレス/射出成形/溶接/表面処理/組立/その他）
materials string[] No 対応素材（アルミ・ステンレス・樹脂等）
certifications string[] No 保有認証（ISO9001, ISO14001, IATF16949等）
equipment_highlight string No 設備アピール（5軸MC、クリーンルーム等）
lot_size enum No 対応ロット（試作1個〜/小ロット/量産）

3. 士業テンプレート

3.1 業種マスター定義

定義項目 値
業種コード PRO
推奨トーン authoritative（専門的だが平易・安心感重視）
一人称 「当事務所」「私たち」
禁止表現 断定的な法的助言（「必ず〜できます」）、報酬の最安保証、他事務所との比較優位表現
E-E-A-T強化要素 資格・登録番号、実務年数、対応件数実績、所属団体
法的注意事項 弁護士法・税理士法・行政書士法等の広告規制を遵守する旨の免責テンプレートを自動付与

3.2 業種固有入力項目

入力項目 型 必須 説明
profession_type enum Yes 士業種別（弁護士/税理士/社労士/行政書士/司法書士）
specialization string[] Yes 専門分野（相続/労務/会社設立/債務整理等）
qualifications string[] Yes 保有資格・登録番号
years_experience number No 実務経験年数
case_count string No 累計相談・対応件数（例: 「年間200件超」）

7. 横断ルール・制約事項

7.1 E-E-A-T準拠チェックリスト（全テンプレート共通）

生成時に以下を自動検証する。未充足項目はユーザーに入力を促す。

E-E-A-T要素 チェック項目 自動/手動
Experience（経験） 実体験・事例の具体性があるか 手動（入力促進）
Expertise（専門性） 資格・経歴・実績が明記されているか 自動（入力済みデータから挿入）
Authoritativeness（権威性） 監修者情報・所属団体が記載されているか 自動（プロフィール枠生成）
Trustworthiness（信頼性） 情報源の明示・免責事項があるか 自動（免責テンプレート挿入）

7.2 SEO共通仕様

項目 仕様
タイトルタグ 30〜35字、主要キーワードを前方配置
メタディスクリプション 100〜120字、行動喚起を含む
見出し構造 H1×1 → H2×3〜5 → H3は必要時のみ
内部リンク 関連記事への自動リンク提案（2〜3本）
構造化データ FAQ / HowTo / LocalBusiness / Product を業種・記事型に応じて自動付与
画像ALT 「{商品名/技術名}の{状態/用途}」形式で自動生成

7.3 トーン定義マスター

トーンID 説明 使用業種
professional 堅実・信頼感。敬体。技術用語を正確に使用。 製造業
authoritative 専門性と安心感。丁寧語。法令用語は平易に言い換え。 士業
warm 五感表現豊富。語りかけ調。絵文字は最小限に。 飲食業
expert_friendly 専門性を保ちつつ非技術者にも伝わる。図表多用。 IT業
approachable 親しみやすく購買意欲を喚起。絵文字・記号を適度に活用。 小売業

8. テンプレート拡張ガイドライン

8.1 新業種追加時の手順

  1. 業種コードを決定（アルファベット3文字）
  2. 業種マスター定義を作成（トーン・禁止表現・E-E-A-T要素）
  3. 業種固有入力項目を設計（5〜8項目が目安）
  4. 4コンテンツタイプのテンプレートを作成
  5. バリエーション展開ルールを最低3軸定義
  6. E-E-A-Tチェックリストとの整合性確認
