1. サービス概要

1.1 サービス名称

AdCraft AI（アドクラフト エーアイ）

1.2 提供価値

AdCraft AIは、EC事業者・D2Cブランドが抱える「広告クリエイティブの量産・最適化コスト」を解消するSaaS型サービスである。従来、1本あたり3〜5万円・納期3〜5営業日を要していた広告バナー制作を、AIによる自動生成で1本あたり500円以下・最短5分に短縮する。

1.3 対象顧客

セグメント 月間広告費 課題
小規模EC事業者 50万〜150万円 デザイナー不在。テンプレ流用で差別化できない
中堅D2Cブランド 150万〜500万円 クリエイティブ検証の速度が運用改善のボトルネック
小規模広告代理店 担当案件合計300万〜1,000万円 多数クライアントのバナー量産で人件費が圧迫される

1.4 サービス機能一覧

コア機能

  • AIクリエイティブ生成: 商品画像・ブランドガイドライン・訴求軸を入力し、広告バナー・LPヒーロー画像・動画サムネイルを自動生成
  • マルチフォーマット自動変換: 1つの原案からLINE広告（1200×628, 1080×1080）、Instagram広告（1080×1080, 1080×1920）、Google広告（300×250, 728×90, 160×600）の全サイズを一括出力
  • A/Bテスト統合: 生成した複数バリエーションを広告プラットフォームに自動入稿し、CTR・CVRデータを収集。勝ちクリエイティブを自動判定
  • ブランド一貫性エンジン: ロゴ配置ルール、カラーパレット、フォント指定、トーン&マナーをプロファイルとして学習し、すべての生成物に適用

運用支援機能

  • パフォーマンスダッシュボード: クリエイティブ別CTR・CVR・ROAS推移をリアルタイム表示
  • 競合クリエイティブ分析: 同業他社の出稿傾向をAIが分析し、差別化ポイントを提示
  • シーズナルテンプレート: セール・季節イベント用のテンプレートライブラリを提供（毎月追加更新）
  • チーム承認ワークフロー: 生成→レビュー→承認→入稿の一連フローをサービス内で完結

1.5 競合優位性

  1. 生成品質: Stable Diffusion XL + 独自ファインチューニングモデルにより、「AIっぽさ」を排除した広告表現を実現
  2. マルチプラットフォーム一括対応: 他社はSNS特化が多い中、LINE・Instagram・Googleの3媒体を統合カバー
  3. ROAS実証モデル: A/Bテスト機能とダッシュボードにより、導入効果を数値で可視化。「費用対効果が見えない」という従来のクリエイティブ制作の課題を解消

4.2 主要コンポーネント

画像生成エンジン

  • ベースモデル: Stable Diffusion XL 1.0
  • 独自ファインチューニング: 広告バナーデータセット（約50万枚）で追加学習
  • ControlNetによるレイアウト制御: ロゴ・テキスト・商品画像の配置を精密制御
  • 推論基盤: AWS EC2 G5インスタンス（NVIDIA A10G）/ オートスケーリング対応
  • 生成時間: 1枚あたり平均8秒（バッチ処理時は1枚あたり3秒）

マルチフォーマット変換エンジン

  • AWS Lambda上で動作する軽量画像処理パイプライン
  • 入力1枚から全サイズ（最大8フォーマット）を並列生成
  • スマートクロッピング: 被写体検出により自動トリミング位置を最適化
  • テキスト再配置: サイズ比率に応じてコピーの改行・フォントサイズを自動調整

A/Bテスト・最適化エンジン

  • Multi-Armed Bandit（Thompson Sampling）アルゴリズムを採用
  • 最低サンプルサイズ到達後に自動で勝者判定（統計的有意水準 p < 0.05）
  • 勝ちクリエイティブの要素（色彩・レイアウト・コピー）をAIが分析し、次回生成にフィードバック

4.3 インフラストラクチャ

  • クラウド: AWS（東京リージョン ap-northeast-1）
  • コンテナ: ECS Fargate（APIサービス群）
  • GPU: EC2 G5（推論）、オンデマンド＋Spot混在でコスト最適化
  • CDN: CloudFront（生成画像配信）
  • CI/CD: GitHub Actions → ECR → ECS Blue/Greenデプロイ
  • 監視: CloudWatch + Datadog（APM）+ PagerDuty（アラート）

6. データセキュリティ方針

6.1 基本方針

AdCraft AIは、顧客から預かる商品画像・ブランド素材・広告運用データを最重要資産として扱い、業界標準以上のセキュリティ対策を講じる。

6.2 データ分類と取扱い

分類 該当データ 保管期間 暗号化
機密 広告アカウント認証情報、API Key 契約期間中 AES-256（保管時）+ TLS 1.3（通信時）
顧客資産 商品画像、ブランドガイドライン、ロゴ 契約終了後90日 AES-256（S3 SSE-KMS）
運用データ 広告パフォーマンスデータ、A/Bテスト結果 契約終了後90日 AES-256（RDS暗号化）
生成データ AI生成クリエイティブ 契約終了後90日 AES-256（S3 SSE-KMS）
ログ アクセスログ、操作ログ 1年間 AES-256

6.3 アクセス制御

  • 認証: SSO（SAML 2.0 / OpenID Connect）対応。多要素認証（MFA）必須（Enterprise）
  • 認可: RBAC（Role-Based Access Control）による最小権限原則
  • オーナー: 全機能アクセス、課金管理、メンバー管理
  • 編集者: クリエイティブ生成・編集、A/Bテスト設定
  • 閲覧者: ダッシュボード閲覧、レポートダウンロード
  • 社内アクセス: 顧客データへの社内アクセスは原則禁止。サポート対応時のみ、顧客承認の上で一時的にアクセス権を付与（操作ログを全件記録）

6.4 インフラセキュリティ

  • ネットワーク: VPC分離、セキュリティグループによるポート制限、WAF適用
  • 脆弱性管理: 週次の自動脆弱性スキャン（Trivy / Snyk）、四半期の第三者ペネトレーションテスト
  • パッチ管理: 重大脆弱性（CVSS 9.0以上）は24時間以内に対応
  • バックアップ: 日次自動バックアップ（S3 Cross-Region Replication）、RPO 24時間、RTO 4時間

6.5 AI学習データに関するポリシー

  • 顧客データの非利用宣言: 顧客がアップロードした画像・素材を、他顧客向けのAIモデル学習に使用しない
  • 生成物の著作権: AI生成クリエイティブの著作権は顧客に帰属する（利用規約に明記）
  • オプトイン学習: 顧客が明示的に同意した場合のみ、匿名化した上でモデル改善に活用（同意はいつでも撤回可能）

6.6 コンプライアンス

項目 対応状況
個人情報保護法 準拠（プライバシーポリシー公開済み）
ISMS（ISO 27001） サービス開始後12ヶ月以内に取得予定
SOC 2 Type II サービス開始後18ヶ月以内に取得予定
景品表示法 AI生成コピーの誇大広告チェック機能を実装

6.7 インシデント対応

  • 検知: CloudTrail + GuardDuty + 独自異常検知ルール
  • 初動: 検知から30分以内にセキュリティチームが調査開始
  • 通知: 個人データ漏洩の可能性がある場合、72時間以内に顧客へ通知
  • 報告: インシデント収束後5営業日以内に原因分析・再発防止策を書面で報告

6.8 契約終了時のデータ処理

  • 契約終了後90日間はデータを保持（顧客によるダウンロード期間）
  • 90日経過後、全データを完全削除（論理削除＋物理削除）
  • 削除完了証明書を顧客に発行（Enterpriseプランは標準、他プランはリクエストベース）
