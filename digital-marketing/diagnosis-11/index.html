はじめに：AI検索時代のルール変更

Google SGE、Perplexity、ChatGPT検索——ユーザーが「検索」する先がAIに移行しつつある。従来のSEOで上位表示されていても、AIの回答に引用されなければトラフィックはゼロになる。本記事では、AIに「引用元」として選ばれるコンテンツの書き方を実例付きで解説する。

1. 従来SEO vs AEO（Answer Engine Optimization）

比較項目 従来SEO AEO
目的 検索結果で上位表示 AIの回答に引用される
最適化対象 Googleのランキングアルゴリズム LLMの情報抽出ロジック
コンテンツ形式 キーワード密度・内部リンク重視 明確な定義文・構造化データ重視
成果指標 順位・CTR AI引用回数・参照トラフィック
文体 自然なキーワード配置 一文で完結する簡潔な回答

要点： SEOが「見つけてもらう技術」なら、AEOは「引用してもらう技術」。両立が理想だが、AEOではコンテンツの構造と明確さがより重要になる。

2. AIに引用されやすい「定義文の型」

AIは明確で簡潔な定義文を好んで引用する。以下の3つの型を使い分けよう。

型1：「〇〇とは、△△である」型

AEOとは、 AIによる回答エンジン（Perplexity、Google SGEなど）に自社コンテンツを引用させることを目的とした最適化手法である。

型2：「〇〇は△△を指す。具体的には〜」型

構造化データは、 Webページの内容を検索エンジンやAIが理解しやすい形式で記述したメタデータを指す。具体的には、JSON-LD形式でページ内の情報（FAQ、レビュー、商品情報など）をマークアップする。

型3：「〇〇のメリットは以下の3点」型（リスト誘導）

FAQスキーマを実装するメリットは以下の3点である。 1. AI検索での引用確率が向上する 2. リッチリザルトとして検索結果に表示される 3. ユーザーの検索意図に直接回答できる

共通ルール： - 最初の1〜2文で定義を完結させる - 主語と述語を近くに置く - 専門用語には括弧で補足を入れる

3. FAQスキーマの具体的な記述例

FAQスキーマはAI検索に最も効果的な構造化データの一つ。JSON-LD形式での実装例を示す。

{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "AEOとは何ですか？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "AEO（Answer Engine Optimization）とは、AIによる回答エンジンに自社コンテンツを引用させるための最適化手法です。従来のSEOがGoogle検索の順位を対象とするのに対し、AEOはPerplexityやChatGPT検索などのAI回答に選ばれることを目指します。"
      }
    },
    {
      "@type": "Question",
      "name": "FAQスキーマの効果は？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "FAQスキーマを実装すると、Googleリッチリザルトへの表示率が向上し、AIが回答を生成する際の引用元として選ばれやすくなります。"
      }
    }
  ]
}

実装時のポイント： - 回答は80〜150文字で簡潔にまとめる - 1ページあたりFAQは5〜10個が目安 - 質問文はユーザーが実際に検索するフレーズを使う - <head>内または</body>直前に配置する

4. 出典明示のベストプラクティス

AIは「信頼性の高い情報源」を優先的に引用する。出典の示し方がコンテンツの信頼性を左右する。

4つの基本ルール

  1. 一次情報にリンクする — 「〇〇によると」ではなく、公式レポートや論文へ直接リンク
  2. 発行年を明記する — 「（出典：総務省, 2025年）」のように年を必ず入れる
  3. 数値には必ず出典をつける — 裏付けのない数値はAIに無視される
  4. 引用元の権威性を示す — 政府統計、学術論文、業界大手のレポートを優先

良い例・悪い例

悪い例 良い例
数値 「市場規模は急成長中」 「国内AI市場は2兆1,700億円（出典：IDC Japan, 2025年）」
主張 「専門家も推奨している」 「Google Search Central公式ドキュメント（2025年更新）で推奨」
統計 「多くの企業が導入済み」 「Fortune 500企業の62%が導入（出典：Gartner, 2025年）」

5. 実践チェックリスト

記事を公開する前に確認すべき項目：

  • [ ] 主要キーワードの定義文が冒頭2文以内にあるか
  • [ ] FAQスキーマをJSON-LDで実装したか
  • [ ] 数値データに出典（発行元・年）を明記したか
  • [ ] 見出しが「質問形式」または「〇〇とは」形式になっているか
  • [ ] 1段落が3行以内に収まっているか
  • [ ] 箇条書き・表で情報が構造化されているか
  • [ ] メタディスクリプションに定義文を含めたか

まとめ

AI検索に選ばれるための3原則：

  1. 明確に定義する — 曖昧な表現を避け、一文で完結する定義文を書く
  2. 構造化する — FAQスキーマ、見出し階層、箇条書きでAIが解析しやすい構造にする
  3. 信頼性を証明する — 全ての主張・数値に権威ある出典を明示する

従来SEOの知識を土台にしつつ、「AIが引用しやすい形式」を意識することが、これからのコンテンツ制作の基本となる。
